# Claude Code グローバルルール

## 🎯 コア原則
- **簡潔性**: 必要最小限の指示のみ記載
- **自動化**: 繰り返し作業は自動化

## 📋 タスク管理

### TODOリスト
- **3ステップ以上のタスクは必ずTODO作成**
- 各タスクを`in_progress`にしてから作業開始
- 完了したら即座に`completed`に更新

## 💻 コーディング規約

### コメント
- すべてのコードに日本語の行末コメント追加
- 初心者にも理解できる平易な説明

## 📁 ファイル構造
```
project/
├── docs/
│   ├── specification.md    # 仕様書
│   └── todo-archive.md     # 完了TODOのアーカイブ
├── README.md               # プロジェクト説明
└── src/                    # ソースコード
```

## 🚀 ワークフロー
```bash
# 1. タスク開始
TodoWriteでタスクリスト作成

# 2. 作業実施
コード編集・テスト実行

# 3. 完了
「完了」と言うだけで自動処理が走る
```
